# 27 — Licenses

## Policy

Orbiteus accepts only **MIT**, **Apache-2.0**, and **BSD-style** licenses for
runtime dependencies. Copyleft licenses (GPL, AGPL, LGPL) are excluded.

Test-only and dev-only dependencies follow the same rule.

## Why

- Predictable redistribution rights for clients.
- Compatible with both proprietary and open-source downstream usage.
- No ambiguity about derivative works.

## Adding a dependency

Required in the PR description:

1. Library name and version.
2. License (verify on the upstream's official source, not just NPM/PyPI metadata).
3. Justification (what it solves, why a stdlib / existing dep can't).
4. Health check: last release date, open issue count, maintainer responsiveness.
5. ADR if it crosses a stack-defining boundary.

## Audit cadence

- Quarterly review of license metadata.
- `pip-audit` and `npm audit` run on every CI build.
- Dependabot weekly security scan.
